# SPEC.md 與現況系統差異分析（更新版）

> 比對基準：`SPEC.md`（你更新後版本）
> 
> 現況來源：目前 Vercel-native 程式碼（2026-04-18）

## A. 總結判斷

- 目前系統已從「單純 Phase1~5 demo」提升為「含學生/教師主流程、登入、管理入口、Prompt/題庫入口」的可操作版本。
- 但與 `SPEC.md` 的 **完整 10 步驟產品規格**相比，仍有明顯差異。
- 最大落差集中在：
  1. 學生端仍以 `legacy_phase (Phase1~5)` 為主
  2. 10 步驟逐步 UI 與子步驟流程未完整落地
  3. 教師端表格與動作細節仍是「功能等價雛形」，非完整規格版

---

## B. 已對齊項目（相對 SPEC）

### 1) 登入與角色分流

- 已有 `/login`，並按角色導向 `/student`、`/teacher`。
- 已有未登入攔截與角色限制（middleware）。
- 已有登出機制。

### 2) 學生首頁任務流程骨幹

- 已有任務列表、加入討論、歷史紀錄。
- 已有 `CourseDetailModal` 風格的任務詳情確認流程。

### 3) 教師端三大功能入口

- 已有系統管理 / 學習管理 / 課程管理分頁入口。
- 已有帳號管理（含重設密碼）。
- 已有課堂觀察、步驟切換、個人進度檢視。

### 4) 課程管理核心操作

- 已有寫作主題 CRUD。
- 已有開課任務 CRUD。
- 已有組別拖曳與儲存分組。

### 5) Prompt 與題庫入口

- 已有主題層 Prompt/題庫 API 與 UI 入口。
- 已有班級任務層 Prompt 覆蓋 API 與 UI 入口。
- 已反映「班級覆蓋高於主題」的概念（配置層級已拆分）。

---

## C. 高優先差異（仍未對齊）

### 1) 主執行流程仍以 Phase1~5 為主，非完整 10 步驟

- `SPEC.md` 目標：學生完整走 1~10 步。
- 現況：學生主頁仍進入 `legacy_phase`，並可「下一階段」推進到 5 為止。
- 影響：6~10（初稿、分析、潤飾、反思、總結）未成為實際主流程。

### 2) 互動模式 UI 規則未完全按 SPEC 強制

- `SPEC.md`：5/7/10 為無互動（不顯示輸入框）；9 為個人反思（AI 不回覆）。
- 現況：`legacy_phase` 聊天頁延續「所有 Phase 可輸入」，與無互動規則衝突。
- 影響：模式不變量在學生端 UI 層未被強制執行。

### 3) 步驟 1/2 的子步驟提問流程未完整落地

- `SPEC.md`：
  - 步驟 1：1-1~1-5 明確來源（題庫/Prompt）
  - 步驟 2：2-1~2-4 明確來源
- 現況：雖已有 Prompt/題庫配置入口，但未建立完整子步驟狀態機與自動提問節奏。
- 影響：流程可配置，但未完整可執行。

### 4) 教師端「檢視學習進度」資料表欄位仍非規格版

- `SPEC.md` 要求小組名稱、成員名單、小組進度、查看小組對話/個人進度等完整欄位與動作。
- 現況：功能已在，但呈現為簡化面板，非完整規格表格型 UI。

---

## D. 中優先差異

### 5) 寫作主題按鈕語意尚未完整對齊

- `SPEC.md`：學生各步驟可點「寫作主題」查看說明。
- 現況：任務補充資料主要在任務詳情區可見，尚未成為每步驟固定按鈕。

### 6) 步驟 3/4 的文章結構樹完整操作未落地

- `SPEC.md`：
  - 步驟 3：可編輯並儲存結構樹
  - 步驟 4：可看他人結構樹（唯讀）
- 現況：尚無完整結構樹編輯/多人瀏覽模組。

### 7) 步驟 6/8 作文編輯器與版本關聯未落地

- `SPEC.md`：
  - 步驟 6 撰寫初稿
  - 步驟 8 預載步驟 6 內容修改
- 現況：尚未形成完整作文編輯器與版本鏈。

---

## E. 低優先差異

### 8) 規格中的欄位與操作文案仍有差異

- 例如表格欄位名、按鈕命名與規格文字不完全一致。
- 屬於可逐步收斂的 UI 一致性問題。

### 9) Mock data 與正式資料模型仍有距離

- 目前大量功能採 `mock-data.ts`。
- 若要正式對齊規格，需逐步替換成正式資料庫與審計流程。

---

## F. 結論與建議

目前系統已達到「可演示現況版流程 + 管理入口基本可操作」；
但若要正式對齊你最新 `SPEC.md`，建議依下列順序推進：

1. 切換學生主流程為 `spec10`（停止以 `legacy_phase` 為主）。
2. 落地步驟 1/2 子步驟狀態機（題庫/Prompt 自動提問）。
3. 完成 5/7/10 無互動與 9 個人反思 UI 強制規則。
4. 補齊步驟 3/4 結構樹、步驟 6/8 作文編輯器、步驟 7/10 報告頁。
5. 將教師觀課與學習進度頁收斂為規格表格與動作集合。
